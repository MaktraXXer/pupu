Ниже формулировка именно для варианта, где будущие выплаты дисконтируются по текущей рыночной OIS-кривой.

Модель CIR++ генерирует множество сценариев будущей краткосрочной ставки, фактически — сценариев RUONIA/OIS short rate. Для оценки инструментов на ключевую ставку к каждой модельной ставке при необходимости добавляется заданный спред:

KS proxy = модельная RUONIA + basis.

В текущем коде basis = 0, поэтому модельные траектории RUONIA напрямую трактуются как proxy ключевой ставки. Если задать basis = 0.002, вся прогнозная КС будет выше модельной RUONIA на 0,20 процентного пункта.

IRS MID считается как такая фиксированная годовая ставка, при которой приведённая стоимость фиксированной ноги равна приведённой стоимости плавающей ноги. Для каждого будущего месяца берётся средняя по Monte Carlo прогнозная ставка, умножается на долю года 1/12 и дисконтируется по текущему рыночному OIS discount factor. Полученный PV плавающей ноги делится на дисконтированный аннуитет фиксированной ноги. Результат — модельная справедливая фиксированная ставка IRS.

Cap и floor считаются по тем же прогнозным траекториям. Для каждого месяца и каждого сценария:

* cap выплачивает max(ставка − страйк, 0) / 12;
* floor выплачивает max(страйк − ставка, 0) / 12.

Каждый будущий платеж дисконтируется не по случайной траектории, а по единой наблюдаемой сегодня OIS-кривой. Затем выплаты суммируются по сроку и усредняются по всем сценариям. Итог — upfront-премия в процентах от номинала.

Роль спреда между КС и RUONIA простая:

* положительный basis повышает IRS MID;
* повышает стоимость cap;
* снижает стоимость floor;
* на дисконтирование не влияет, потому что дисконтирование всегда идёт по OIS-кривой RUONIA.

То есть в этой версии:

* RUONIA/OIS задаёт и модельные сценарии, и рыночное дисконтирование;
* basis используется только для перехода от RUONIA к proxy ключевой ставке;
* IRS, cap и floor оцениваются из одних и тех же сценариев, но по разным формулам выплат.
