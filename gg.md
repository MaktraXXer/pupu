Да. Эти SUMPRODUCT почти наверняка тяжелее, чем нужно.

По смыслу у тебя тут не “настоящий sumproduct”, а по сути поиск одного значения на пересечении строки и колонки:

* строка ищется по AP
* колонка ищется по заголовку AQ2:AZ2
* потом берется значение из матрицы AQ:AZ

Для такого случая намного быстрее обычно INDEX + MATCH.

1) Первая формула

Было:

=SUMPRODUCT(('Параметры S-curve'!$AQ$3:$AZ$503)*('Параметры S-curve'!$AP$3:$AP$503=ROUND(G15;1))*('Параметры S-curve'!$AQ$2:$AZ$2=V15))

Быстрее:

=INDEX('Параметры S-curve'!$AQ$3:$AZ$503;MATCH(ROUND(G15;1);'Параметры S-curve'!$AP$3:$AP$503;0);MATCH(V15;'Параметры S-curve'!$AQ$2:$AZ$2;0))

⸻

2) Вторая формула

Было:

=IF(C$2>360-$A3;0;SUMPRODUCT(('Параметры S-curve'!$AQ$3:$AZ$503)*('Параметры S-curve'!$AP$3:$AP$503=ROUND('Матрицастимулов'!C3*100;1))*('Параметры S-curve'!$AQ$2:$AZ$2=C$1)))

Быстрее:

=IF(C$2>360-$A3;0;INDEX('Параметры S-curve'!$AQ$3:$AZ$503;MATCH(ROUND('Матрицастимулов'!C3*100;1);'Параметры S-curve'!$AP$3:$AP$503;0);MATCH(C$1;'Параметры S-curve'!$AQ$2:$AZ$2;0)))

⸻

Когда это корректно

Эта замена корректна, если:

* в 'Параметры S-curve'!$AP$3:$AP$503 для каждого значения только одна строка;
* в 'Параметры S-curve'!$AQ$2:$AZ$2 каждый заголовок встречается только один раз.

Если там есть дубли, SUMPRODUCT их суммирует, а INDEX+MATCH возьмет только первое совпадение.

⸻

Если возможны дубли

Тогда лучше оставить суммирование, но сделать его быстрее через SUMIFS по отдельным столбцам нельзя удобно, потому что у тебя выбор столбца тоже динамический.

В таком случае можно сделать промежуточный MATCH колонки и потом суммировать только один столбец, а не весь блок AQ:AZ.

Например для первой:

=SUMIFS(INDEX('Параметры S-curve'!$AQ$3:$AZ$503;0;MATCH(V15;'Параметры S-curve'!$AQ$2:$AZ$2;0));'Параметры S-curve'!$AP$3:$AP$503;ROUND(G15;1))

И для второй:

=IF(C$2>360-$A3;0;SUMIFS(INDEX('Параметры S-curve'!$AQ$3:$AZ$503;0;MATCH(C$1;'Параметры S-curve'!$AQ$2:$AZ$2;0));'Параметры S-curve'!$AP$3:$AP$503;ROUND('Матрицастимулов'!C3*100;1)))

Это тоже обычно быстрее, чем полный SUMPRODUCT, потому что не перемножается вся двумерная матрица.

⸻

Что я бы советовал

Если дубликатов нет — ставить именно это:

=INDEX('Параметры S-curve'!$AQ$3:$AZ$503;MATCH(ROUND(G15;1);'Параметры S-curve'!$AP$3:$AP$503;0);MATCH(V15;'Параметры S-curve'!$AQ$2:$AZ$2;0))

и

=IF(C$2>360-$A3;0;INDEX('Параметры S-curve'!$AQ$3:$AZ$503;MATCH(ROUND('Матрицастимулов'!C3*100;1);'Параметры S-curve'!$AP$3:$AP$503;0);MATCH(C$1;'Параметры S-curve'!$AQ$2:$AZ$2;0)))

Это самый быстрый и чистый вариант.

Если хочешь, я могу следующим сообщением дать еще более быстрый вариант через вспомогательные MATCH в отдельных ячейках, чтобы Excel не пересчитывал одни и те же MATCH тысячи раз по матрице.
